# Todorio — Техническое задание (v1)

<aside>
⚡

**Todorio** — приватный self-hosted менеджер задач для личной работы и небольших команд. Ставится на VPS одной командой, работает без внешних SaaS и почты. Главная фишка — **«Пульс пространства»**.

</aside>

## 1. Обзор

**Название:** Todorio

**Слоган:** Your private workspace for tasks and teams.

**Тип:** self-hosted веб-приложение (устанавливается на собственный VPS).

**Аудитория:** личное использование и небольшие команды.

**Репозиторий:** `github.com/DanMotive/Todorio` · **Лицензия:** Apache 2.0

**Ключевые принципы:**

- приватность: все данные на своём сервере, без внешних сервисов;
- без email и SMTP (порт 25 не требуется);
- нет публичного API и API-токенов — только собственный интерфейс;
- удобная установка и управление через веб-панель и терминал;
- лёгкость на VPS, красивый и цельный интерфейс.

## 3. Установка и развёртывание

Установка одной командой с GitHub (фиксированный тег релиза + проверка контрольных сумм):

```bash
curl -fsSL https://raw.githubusercontent.com/DanMotive/Todorio/v1.0.0/scripts/install.sh | sudo bash
sudo todorio setup
```

Скрипт: проверяет ОС (Ubuntu/Debian) и архитектуру, скачивает бинарник из GitHub Releases, проверяет SHA-256, ставит бинарник, запускает `setup`.

### Интерактивный `setup`

При установке спрашивает:

- логин root-администратора (обязательный, уникальный);
- **генерирует временный пароль 16 символов** (буквы, цифры, спецсимволы), показывает один раз, не пишет в логи; в БД `must_change_password = true`;
- менеджер процессов: **systemd (по умолчанию)** / Docker Compose / PM2;
- порт сайта (проверка: 1–65535, не занят, при необходимости открыть в UFW);
- HTTPS с self-signed сертификатом (да/нет);
- создать обучающее демо-пространство с заданиями-квестами (y/n).

Поддержка неинтерактивного режима флагами (`--root-username`, `--process-manager`, `--port`, `--generate-password`).

### Доступ по IP и SSL

- Сайт работает по `http://<IP>` без домена (backend слушает `0.0.0.0`).
- Опционально self-signed SSL по IP (шифрование, но предупреждение браузера).
- Для доверенного сертификата (Let's Encrypt) нужен домен.

### Обслуживание

```bash
sudo systemctl status todorio
sudo todorio backup create
sudo todorio doctor
sudo todorio update
```

`todorio doctor` проверяет: сервис, БД, свободное место, хранилище вложений, срок SSL, актуальность миграций, настроен ли бэкап.

## 4. Роли и права (RBAC)

| Роль | Возможности |
| --- | --- |
| **Root admin** | Всё: пользователи, роли, права, политики, аудит, данные, брендинг, шаблоны |
| **Admin** | Создание пользователей и списков, управление обычными участниками (не root) |
| **User** | Свои списки и те, к которым дан доступ |
| **Viewer** | Только просмотр назначенного |
- Root-админ ровно один, создаётся при `setup`, не может быть удалён или лишён прав через веб.
- Роль — базовый уровень прав; отдельные разрешения тонко настраиваются при одобрении пользователя.

### Статусы пользователя

```
pending   — зарегистрировался, ждёт обработки
active    — одобрен, может работать
blocked   — доступ отключён
rejected  — регистрация отклонена
```

### Регистрация с ручным одобрением

Пользователь регистрируется, но до одобрения root/admin доступны только: страница ожидания, смена своего пароля, выход. Нельзя создавать списки, видеть пользователей, задачи, шаблоны, загружать файлы.

При одобрении админ задаёт роль и точечные разрешения: создавать личные списки, создавать общие списки, загружать файлы и т. д.

### Авторизация

- вход по логину и паролю (без email);
- опциональный TOTP (Google Authenticator / Aegis), особенно для root;
- восстановление доступа — сброс пароля админом (без писем);
- при первом входе — принудительная смена временного пароля.

## 5. Пространства, списки и задачи

### Пространства (группы)

Верхний уровень организации. Содержат списки и заметки, имеют участников и роли.

```
Сервер
└── Пространства
    ├── Личное пространство Vlad
    │   ├── Покупки
    │   └── Идеи
    └── Danmotive
        ├── Разработка
        ├── Инфраструктура
        └── Релизы
```

- у пространства свои участники, роли, workflow, поля, метки, шаблоны, объявления, статистика;
- количество/право создания пространств задаётся политиками root;
- внутри общего пространства возможен приватный список (доступ только автору).

### Списки

- личный список: по умолчанию виден только автору;
- общий список: участники с правами `owner` / `editor` / `viewer`;
- личный список технически = список с одним участником-владельцем.

### Задачи

Поля задачи: название, описание, статус, приоритет, дедлайн, исполнитель, наблюдатели, теги/метки, чек-лист, подзадачи, вложения, прогресс, пользовательские поля, история.

- быстрый ввод с распознаванием: `Купить домен #infra !high @Vlad tomorrow`;
- подзадачи с автоматическим прогрессом родителя;
- зависимости (задача заблокирована другой);
- повторяющиеся задачи (после выполнения создаётся следующая);
- режим «на проверке» (владелец принимает / возвращает).

### Настраиваемые статусы (workflow)

Настраивает создатель пространства, например:

```
Идея → Новая → В работе → На проверке → Готово
Запланировано → Выполняется → Отложено → Отменено → Завершено
```

- текст, `@упоминания`, редактирование/удаление своих, ответы (после v1);
- системные записи в ленте (смена статуса, дедлайна, исполнителя).

- к задачам и комментариям (аватары — позже);

Упоминания, комментарии, реакции, назначение исполнителем, изменение дедлайна/статуса, просрочки. Каждый тип включается/выключается в профиле. Всё внутри сайта, без почты.

## 9. Локализация

### Формат locale: язык-страна

```
en-US · ru-RU · uk-UA · be-BY · kk-KZ
es-ES · pt-BR · tr-TR
zh-CN · hi-IN · bn-BD · ja-JP · ko-KR
```

### IT-стили (сленг)

Отдельные пакеты фраз поверх локали: `ru-RU-it`, `en-US-it`. Пример: «Новая задача» → «Новый таск», «Выполнено» → «Done». В админке — обычный стиль во избежание двусмысленности.

### Автовыбор языка

1. язык из профиля (главный);
2. иначе `Accept-Language` браузера;
3. подбор регионального варианта (`ru → ru-RU`, `pt → pt-BR`);
4. fallback — системный язык сервера (`en-US`).

Строки интерфейса — в отдельных файлах `web/locales/*.json`. Даты/числа/относительное время локализуются. Названия списков/задач/комментариев не переводятся.

## 10. Настройки, лимиты и политики

Всё настраивается **и в root-панели, и в терминале** (единый источник настроек).

### Лимиты (пример)

```
Файлы: разрешены, типы JPG/PNG/WEBP/GIF, размер 10 MB,
       10 на задачу, 5 на комментарий, хранилище 20 GB, 1 GB на пользователя
Контент: списков на пользователя 100, задач в списке 10 000,
         комментариев в задаче 1 000, длина комментария 10 000 символов
Безопасность: попыток входа 10, блокировка 15 мин, сессий на пользователя 10
```

`0` = «без ограничений» (в UI показывать «Без лимита»).

### CLI

```bash
todorio server config set default_locale ru-RU
todorio server config set detect_browser_locale true
todorio server locales enable tr-TR
todorio server limits set uploads.max_file_size_mb 25
todorio server policy set registration.mode invite_only
todorio server policy set users.can_create_spaces false
todorio server branding set site_name "Todorio"
```

### Иерархия настроек

```
Root-админ → глобальные границы и политики сервера
  └── Создатель пространства → настройки своего пространства
      └── Владелец списка → список в разрешённых рамках
```

Root может отключить функцию глобально, но не микроменеджит каждое пространство.

### Политики (примеры)

- регистрация: закрыта / по приглашениям / открыта с одобрением;
- кто создаёт пространства, общие списки, приглашает участников;
- публичные read-only ссылки (по умолчанию выключены, опционально с паролем);
- включены ли комментарии, реакции, вложения, версии, статистика;
- порог сводки после отсутствия (например, 6 часов).

### Подтверждения и защита

Подтверждение перед: удалением пространства, массовым удалением задач, удалением вложений, сбросом прав. Лимиты действий: не более N задач/файлов за период, ограничение попыток входа.

## 11. Архив и версии

- **Архив:** обычное удаление = перемещение в архив (задачи, списки, пространства, пользователи). Автоочистка через **30 дней** простоя; восстановление сбрасывает срок; предупреждение за 3 дня.
- **Полное удаление:** отдельно, только root, с подтверждением.
- **История изменений:** журнал действий у задачи, списка, пространства.
- **Версии задачи:** история содержимого с просмотром изменений и восстановлением версии.

## 12. Представления и продуктивность

- представления: Список, Kanban, Календарь, Таймлайн, Таблица, «Моя неделя»;
- сохранённые фильтры;
- «Входящие» и быстрый ввод;
- избранное / закреплённые;
- фокус-режим с таймером и учётом времени;
- страница **«Моё»** (сегодня, просрочено, на проверке, без дедлайна, упоминания);
- горячие клавиши (работают вне полей ввода, отключаемы, `?` — подсказка, опасные действия без односимвольных клавиш);
- полноценный поиск по доступному контенту (задачи, заметки, комментарии) с фильтрами;
- «Не беспокоить» с отложенными уведомлениями;
- заметки (Markdown-страницы внутри пространства/списка);
- **PWA:** установка сайта как приложения (иконка, полноэкранный режим), кнопка «Установить приложение» в меню; требует HTTPS (self-signed подходит после принятия сертификата);
- **онбординг-квесты:** обучающие задания для нового пользователя после одобрения («создай задачу», «поставь дедлайн», «оставь реакцию»…) с прогресс-баром освоения; отключаются;
- **звук уведомлений:** тихий сигнал на упоминания/назначения, выключен по умолчанию, настраивается в профиле; работает в открытой вкладке без домена (системные push-уведомления браузера — только при HTTPS).

## 13. Пользовательские поля и метки

**Пользовательские поля** (настраивает создатель пространства): текст, число, дата, выбор, мультивыбор, чекбокс, пользователь, ссылка, рейтинг. Разные наборы для разных пространств.

**Метки/категории** — цветные ярлыки для группировки и фильтров (реализуются как поле типа «выбор», чтобы не плодить две системы).

## 14. Статистика, подписи и рейтинги

### Личная статистика

```
За июль: закрыто 47 · вовремя 79% · просрочено 6 · самый активный список · время в фокусе
```

### Динамические подписи (из двух частей)

~25 вариантов на каждую часть, с категориями (`success`, `perfect_day`, `overdue`, `inactive`, `focus`, `project`, `neutral`), чтобы не было бессмыслицы. Выбор фиксируется на дату+пользователя. Для каждого языка — свои естественные наборы, не машинный перевод. Стиль: нейтральный / дружелюбный / IT / отключено.

```json
{
  "stats.success.first": ["Сегодня закрыто {count} задач.", "За сегодня выполнено {count} задач."],
  "stats.success.second": ["Хороший темп.", "Бэклог стал легче."]
}
```

### Рейтинги в пространстве

Опциональны, настраивает владелец пространства: период, что учитывать (закрытые задачи, в срок, комментарии, реакции, фокус), кому показывать (своё место / топ-3 / полная таблица / только владельцу). Взвешенный балл (учёт сложности и веса), чтобы не поощрять дробление задач.

### Сводка после отсутствия

Показывается при возвращении после N часов (по умолчанию 6, настраивается). Что произошло за время отсутствия.

### Лента активности

Отдельная страница/блок по доступным пространствам.

## 17. ⭐ Пульс пространства (главная фишка)

Живая сводка состояния проекта — что движется, где риск, куда смотреть. Считается по фактам (без ИИ и внешнего API).

```
Пульс: Todorio                         72%  🟢

Внимание:
🔴 2 задачи просрочены
🟠 «Настроить SSL» без обновлений 4 дня
🟡 3 задачи без исполнителя
⛔ «Выпуск релиза» заблокирован задачей «Собрать пакет»

Сейчас в работе:
• Авторизация — Alex — 65%

Следующее лучшее действие:
→ Назначить исполнителя для «Проверить бэкапы»
```

Сигналы: просрочка, давно нет изменений, заблокировано, нет исполнителя, нет дедлайна, прогресс стоит, ждёт проверки, давно нет активности.

Настройки владельца пространства: какие сигналы показывать, порог «неактивности», пороги состояния 🟢🟡🔴. Root включает/выключает функцию глобально.

Дополняет: ежедневный мини-стендап («что сделал / делаю / что блокирует»).

- светлая и тёмная темы (тёмная по умолчанию для технических пространств);
- **5 цветовых тем:** красная, синяя, зелёная, жёлтая, серая (акцентный цвет всего интерфейса);
- root задаёт тему, цвет и визуальный режим по умолчанию для сервера; каждый пользователь может сменить их в своём профиле;
- **два визуальных режима:** «красивый» (прозрачности, glassmorphism, плавные анимации — требовательнее к устройству) и «лёгкий» (минимум эффектов, быстрый на слабом железе; уважает prefers-reduced-motion);
- аккуратные семантические цвета статусов;
- заметные, но спокойные прогресс-бары и «Пульс»;
- быстрые анимации, полноценная мобильная версия;
- с самого начала: палитра, типографика, отступы, иконки, состояния кнопок/карточек/таблиц.

---

## 21. Что НЕ входит

- публичный API и API-токены;
- email/SMTP, письма, ссылки-приглашения по почте;
- webhooks и внешние интеграции (в т. ч. Telegram) — возможно позже;
- гостевая роль отдельного вида (признано избыточным);
- рабочие пространства как отдельная сущность сверх групп — реализуются как пространства.
